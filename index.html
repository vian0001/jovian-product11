1. PENGENALAN SERVICE IP PADA MIKROTIK

Tampilan halaman SERVICE IP pada MikroTik RouterOS

Dari sumber NetworkTik (2024), MikroTik RouterOS memiliki beberapa SERVICE yang secara default aktif untuk tujuan administrasi. SERVICE-SERVICE ini memungkinkan administrator untuk mengakses dan mengelola router melalui berbagai protokol dan metode.

Untuk melihat SERVICE yang tersedia pada RouterOS, gunakan perintah berikut:

/ip service print
# Menampilkan daftar semua SERVICE yang tersedia

Dari sumber System Zone (2021), secara default MikroTik RouterOS mengaktifkan beberapa SERVICE untuk mengakses RouterOS untuk tujuan administrasi. SERVICE-SERVICE ini meliputi:

  • API - SERVICE untuk akses program eksternal menggunakan RouterOS API
  • FTP - SERVICE untuk transfer file ke dan dari router
  • SSH - SERVICE untuk akses command line aman
  • Telnet - SERVICE untuk akses command line (tidak aman)
  • Winbox - SERVICE untuk aplikasi manajemen berbasis GUI Winbox
  • WWW - SERVICE untuk akses WebFig (antarmuka web)
  • WWW-SSL - SERVICE untuk akses WebFig melalui HTTPS
  • API-SSL - SERVICE untuk akses API melalui koneksi aman

Dari sumber MikroTik Documentation (2023), setiap SERVICE memiliki port default yang ditugaskan sebagai berikut:

# Port default untuk SERVICE MikroTik RouterOS
api - port 8728
api-ssl - port 8729
ftp - port 21
ssh - port 22
telnet - port 23
winbox - port 8291
www - port 80
www-ssl - port 443

Dari sumber NetworkTik (2024), setiap SERVICE di MikroTik RouterOS memiliki properti konfigurasi yang sama termasuk:

  • name - Nama SERVICE
  • port - Port TCP yang digunakan SERVICE
  • address - Alamat IP atau subnet yang diizinkan mengakses SERVICE
  • certificate - Sertifikat untuk SERVICE berbasis SSL
  • disabled - Status aktivasi SERVICE
  • tls-version - Versi TLS yang diizinkan (untuk SERVICE berbasis SSL)

Kesimpulan: Dari 3 pendapat tersebut dapat disimpulkan bahwa MikroTik RouterOS menyediakan berbagai SERVICE IP untuk keperluan administrasi dan manajemen router. Setiap SERVICE memiliki fungsi spesifik, port default, dan properti konfigurasi yang dapat disesuaikan. Memahami SERVICE-SERVICE ini sangat penting untuk pengelolaan router secara efektif dan aman.

2. MENGELOLA SERVICE IP

Tampilan konfigurasi SERVICE IP pada MikroTik RouterOS

Dari sumber NetworkTik (2024), mengelola SERVICE IP pada MikroTik RouterOS sangat penting untuk meningkatkan keamanan router. Praktik terbaik keamanan mengharuskan menonaktifkan SERVICE yang tidak digunakan untuk mengurangi permukaan serangan.

Untuk menonaktifkan SERVICE yang tidak digunakan, gunakan perintah berikut:

/ip service disable telnet,ftp
# Menonaktifkan SERVICE telnet dan ftp

Atau dengan menentukan nomor SERVICE:

/ip service disable numbers=1,2
# Menonaktifkan SERVICE dengan nomor 1 dan 2

Dari sumber MikroTik Documentation (2023), untuk mengaktifkan kembali SERVICE yang telah dinonaktifkan:

/ip service enable ssh
# Mengaktifkan SERVICE SSH

Untuk mengubah port default SERVICE dan meningkatkan keamanan:

/ip service set ssh port=2222
# Mengubah port SSH dari 22 menjadi 2222
/ip service set www port=8081
# Mengubah port HTTP dari 80 menjadi 8081
/ip service set winbox port=9090
# Mengubah port Winbox dari 8291 menjadi 9090

Dari sumber System Zone (2021), administrator jaringan juga dapat membatasi SERVICE agar hanya dapat diakses dari alamat IP atau subnet tertentu:

/ip service set ssh address=192.168.88.0/24
# Membatasi akses SSH hanya dari jaringan 192.168.88.0/24
/ip service set winbox address=192.168.88.0/24,10.0.0.5
# Membatasi akses Winbox dari jaringan 192.168.88.0/24 dan alamat 10.0.0.5

Dari sumber MikroTik Documentation (2024), SERVICE telnet dan ftp sebaiknya dinonaktifkan karena mengirim kredensial dalam teks biasa (tidak terenkripsi). Sebagai gantinya, gunakan SSH untuk akses command line dan SFTP untuk transfer file. Jika memerlukan akses berbasis web, gunakan www-ssl (HTTPS) daripada www (HTTP) untuk enkripsi komunikasi.

Untuk menyetel SERVICE www-ssl dengan sertifikat:

/ip service set www-ssl certificate=my-cert disabled=no
# Mengaktifkan HTTPS dengan sertifikat tertentu

Dari sumber System Zone (2021), beberapa administrator MikroTik RouterOS memodifikasi port SERVICE login dengan port yang tidak digunakan. Namun ini memiliki kerugian karena Anda mungkin lupa nomor port yang dimodifikasi dan tidak dapat masuk ke Router MikroTik sampai mengingat nomor port yang ditetapkan. Pastikan untuk mendokumentasikan perubahan port dengan baik.

Kesimpulan: Dari 4 pendapat tersebut dapat disimpulkan bahwa pengelolaan SERVICE IP pada MikroTik RouterOS melibatkan aktivasi/deaktivasi SERVICE, mengubah port default, dan membatasi akses berdasarkan alamat IP. Praktik terbaik keamanan mengharuskan menonaktifkan SERVICE yang tidak digunakan (terutama yang tidak terenkripsi seperti telnet dan ftp), mengubah port default, dan membatasi akses ke SERVICE hanya dari jaringan yang tepercaya.

3. SERVICE API DAN API-SSL

Dari sumber MikroTik Documentation (2024), API (Application Programming Interface) MikroTik memungkinkan pengembang untuk membuat aplikasi yang dapat berkomunikasi dengan RouterOS. API MikroTik menggunakan protokol berbasis teks sederhana yang disebut protokol API RouterOS.

RouterOS menyediakan dua SERVICE untuk akses API:

  • api - SERVICE API tanpa enkripsi (port default 8728)
  • api-ssl - SERVICE API dengan enkripsi SSL/TLS (port default 8729)

Untuk memeriksa status SERVICE API:

/ip service print where name="api" || name="api-ssl"

Dari sumber RouterOS API Documentation (2023), untuk mengonfigurasi SERVICE API:

# Mengaktifkan API dengan pembatasan alamat IP
/ip service set api address=192.168.88.0/24 disabled=no
# Mengaktifkan API-SSL dengan sertifikat
/ip service set api-ssl certificate=my-cert address=192.168.88.0/24 disabled=no

Dari sumber MikroTik API Tutorial (2022), SERVICE API memungkinkan berbagai fungsi otomatisasi dan integrasi, termasuk:

  • Pemantauan perangkat MikroTik secara terpusat
  • Otomatisasi perubahan konfigurasi pada banyak perangkat
  • Integrasi dengan sistem pemantauan jaringan
  • Pembuatan antarmuka pengguna kustom
  • Pengembangan alat administrasi jaringan

Contoh sederhana skrip API menggunakan Python:

# Contoh skrip Python menggunakan API MikroTik
import routeros_api
connection = routeros_api.RouterOsApiPool('192.168.88.1',
username='admin',
password='password',
port=8728,
plaintext_login=True)
api = connection.get_api()
list_address = api.get_resource('/ip/address')
addresses = list_address.get()
for address in addresses:
print(address)

Dari sumber Network Security Best Practices (2023), SERVICE API tanpa SSL mengirimkan kredensial dan data dalam bentuk tidak terenkripsi. Untuk keamanan yang lebih baik, selalu gunakan api-ssl daripada api, terutama jika API diakses melalui jaringan yang tidak tepercaya. Batasi akses API hanya ke alamat IP administrator yang tepercaya.

Kesimpulan: Dari 4 pendapat tersebut dapat disimpulkan bahwa SERVICE API dan API-SSL MikroTik menyediakan antarmuka yang kuat untuk otomatisasi dan integrasi dengan sistem lain. API-SSL memberikan keamanan tambahan melalui enkripsi dan harus digunakan daripada API biasa ketika memungkinkan. Membatasi akses API hanya ke alamat IP tertentu juga merupakan praktik keamanan penting untuk mencegah akses tidak sah.

4. SERVICE WWW DAN WWW-SSL

Tampilan WebFig (antarmuka web) MikroTik RouterOS

Dari sumber MikroTik Documentation (2024), SERVICE www dan www-ssl memungkinkan akses ke antarmuka web MikroTik RouterOS yang disebut WebFig. WebFig menyediakan antarmuka grafis berbasis web untuk mengelola router tanpa memerlukan aplikasi Winbox.

RouterOS menyediakan dua SERVICE untuk akses web:

  • www - SERVICE HTTP tanpa enkripsi (port default 80)
  • www-ssl - SERVICE HTTPS dengan enkripsi SSL/TLS (port default 443)

Untuk memeriksa status SERVICE web:

/ip service print where name="www" || name="www-ssl"

Dari sumber RouterOS WebFig Guide (2023), untuk mengonfigurasi SERVICE www-ssl dengan sertifikat:

# Membuat sertifikat self-signed
/certificate add name=webfig-cert common-name=router.local key-usage=tls-server
/certificate sign webfig-cert
# Mengaktifkan www-ssl dengan sertifikat
/ip service set www-ssl certificate=webfig-cert disabled=no
# Menonaktifkan www (HTTP) untuk keamanan
/ip service set www disabled=yes

Dari sumber MikroTik Web Access Security (2022), WebFig menyediakan hampir semua fungsi yang tersedia di Winbox, termasuk:

  • Konfigurasi interface jaringan
  • Manajemen routing
  • Konfigurasi firewall dan NAT
  • Manajemen pengguna dan grup
  • Pemantauan sistem dan jaringan
  • Manajemen SERVICE dan protokol

Untuk membatasi akses ke SERVICE web berdasarkan alamat IP:

/ip service set www-ssl address=192.168.88.0/24,10.0.0.5
# Membatasi akses HTTPS hanya dari jaringan 192.168.88.0/24 dan alamat 10.0.0.5

Dari sumber Network Security Best Practices (2023), SERVICE www (HTTP) mengirimkan kredensial dan data dalam bentuk tidak terenkripsi yang dapat dicegat dan dibaca oleh pihak tidak berwenang. Untuk keamanan yang lebih baik, gunakan www-ssl (HTTPS) yang mengenkripsi semua lalu lintas antara browser dan router. Jika memungkinkan, nonaktifkan SERVICE www dan hanya gunakan www-ssl.

Dari sumber MikroTik TLS Configuration (2023), untuk meningkatkan keamanan www-ssl dengan membatasi versi TLS:

/ip service set www-ssl tls-version=only-1.2,only-1.3
# Membatasi www-ssl hanya menggunakan TLS v1.2 dan v1.3 (yang lebih aman)

Kesimpulan: Dari 5 pendapat tersebut dapat disimpulkan bahwa SERVICE www dan www-ssl MikroTik menyediakan antarmuka web untuk administrasi router melalui WebFig. Untuk keamanan, sebaiknya menggunakan www-ssl dengan sertifikat yang valid dan menonaktifkan SERVICE www biasa. Selain itu, membatasi akses berdasarkan alamat IP dan menggunakan versi TLS terbaru juga merupakan praktik keamanan penting untuk melindungi akses web ke router.

5. SERVICE WINBOX

Aplikasi Winbox untuk manajemen MikroTik RouterOS

Dari sumber MikroTik Documentation (2024), Winbox adalah utilitas administrasi berbasis GUI Windows yang digunakan untuk mengonfigurasi MikroTik RouterOS. SERVICE winbox di menu IP Service mengontrol akses ke protokol Winbox pada router.

Dari sumber Winbox Security Guide (2023), SERVICE Winbox menggunakan port TCP 8291 secara default dan menyediakan antarmuka grafis yang kaya fitur untuk mengelola router MikroTik. Winbox dapat terhubung ke router melalui:

  • IP Address - Koneksi standar melalui jaringan IP
  • MAC Address - Koneksi langsung melalui Layer 2 (harus berada di jaringan lokal yang sama)

Untuk memeriksa status SERVICE Winbox:

/ip service print where name="winbox"

Untuk mengonfigurasi SERVICE Winbox:

# Mengubah port Winbox
/ip service set winbox port=9090
# Membatasi akses Winbox berdasarkan alamat IP
/ip service set winbox address=192.168.88.0/24,10.0.0.5

Dari sumber RouterOS Mac Access Guide (2022), akses Winbox melalui MAC dapat dinonaktifkan untuk keamanan tambahan. Ini dapat dilakukan melalui:

# Menonaktifkan akses Winbox via MAC
/tool mac-server set allowed-interface-list=none
/tool mac-server mac-winbox set allowed-interface-list=none

Dari sumber MikroTik Security Recommendations (2023), Winbox mengenkripsi kata sandi selama transmisi tetapi bukan seluruh sesi. Untuk akses jarak jauh yang aman, sebaiknya gunakan Winbox melalui koneksi VPN atau SSH tunnel. Selain itu, sebaiknya ubah port default Winbox dan batasi akses hanya ke alamat IP administrator yang tepercaya.

Dari sumber Winbox Advanced Configuration (2024), komunikasi Winbox dapat dienkripsi menggunakan fitur "Secure Mode" di aplikasi Winbox. Ketika menggunakan mode ini, pastikan SERVICE winbox di router dikonfigurasi dengan benar.

Untuk mengakses Winbox secara jarak jauh dengan aman melalui SSH tunnel:

# Pada komputer Windows dengan Putty:
1. Buat koneksi SSH ke router MikroTik
2. Di Putty, konfigurasi tunneling: Source port 9090, Destination 127.0.0.1:8291
3. Buka Winbox dan hubungkan ke 127.0.0.1:9090

Kesimpulan: Dari 5 pendapat tersebut dapat disimpulkan bahwa SERVICE Winbox adalah metode administrasi MikroTik yang populer karena antarmuka grafisnya yang intuitif. Untuk keamanan, praktik terbaik meliputi mengubah port default, membatasi akses berdasarkan alamat IP, menonaktifkan akses MAC jika tidak diperlukan, dan menggunakan SSH tunnel untuk akses jarak jauh. Meskipun Winbox mengenkripsi kata sandi, tidak semua lalu lintas terenkripsi, sehingga harus digunakan dengan hati-hati di jaringan yang tidak aman.

6. SERVICE SSH, TELNET, DAN FTP

Terminal SSH untuk mengakses MikroTik RouterOS

Dari sumber MikroTik Documentation (2024), RouterOS menyediakan beberapa SERVICE untuk akses command line dan transfer file:

  • SSH - Akses command line terenkripsi (port default 22)
  • Telnet - Akses command line tidak terenkripsi (port default 23)
  • FTP - Transfer file tidak terenkripsi (port default 21)

Untuk memeriksa status SERVICE ini:

/ip service print where name="ssh" || name="telnet" || name="ftp"

Dari sumber RouterOS CLI Access Guide (2023), SSH (Secure Shell) adalah metode yang direkomendasikan untuk akses command line karena mengenkripsi semua lalu lintas termasuk kredensial login. Untuk mengonfigurasi SSH:

# Mengaktifkan SSH dengan port yang diubah
/ip service set ssh port=2222 address=192.168.88.0/24 disabled=no
# Mengonfigurasi algoritma enkripsi SSH
/ip ssh set strong-crypto=yes

Dari sumber Network Security Best Practices (2023), SERVICE telnet dan ftp harus dinonaktifkan karena mengirimkan semua data, termasuk nama pengguna dan kata sandi, dalam bentuk teks biasa yang tidak terenkripsi. Gunakan SSH sebagai pengganti telnet dan SFTP (yang berjalan melalui SSH) sebagai pengganti FTP. Untuk menonaktifkan SERVICE yang tidak aman:

/ip service set telnet disabled=yes
/ip service set ftp disabled=yes

Dari sumber SSH Key Authentication Guide (2022), untuk meningkatkan keamanan SSH, Anda dapat menggunakan otentikasi kunci publik daripada kata sandi:

# Mengimpor kunci publik SSH untuk pengguna
/user ssh-keys import user=admin public-key-file=id_rsa.pub

Dari sumber MikroTik CLI Scripting (2023), SSH juga memungkinkan eksekusi skrip jarak jauh untuk otomatisasi:

# Contoh mengeksekusi perintah jarak jauh melalui SSH
ssh admin@192.168.88.1 "/system resource print"

Untuk transfer file dengan aman menggunakan SFTP (melalui SSH):

# Pada komputer Linux/Mac
sftp admin@192.168.88.1
get backup.backup
put config.rsc
# Atau menggunakan aplikasi SFTP seperti FileZilla, WinSCP, dll.

Kesimpulan: Dari 5 pendapat tersebut dapat disimpulkan bahwa SSH adalah SERVICE yang direkomendasikan untuk akses command line dan transfer file ke MikroTik RouterOS, sementara telnet dan FTP sebaiknya dinonaktifkan karena masalah keamanan. Penggunaan otentikasi kunci SSH daripada kata sandi, mengubah port default, dan membatasi akses berdasarkan alamat IP adalah praktik terbaik untuk mengamankan akses SSH. SSH juga memungkinkan otomatisasi dan eksekusi skrip jarak jauh yang berguna untuk administrasi jaringan.

7. PENGUJIAN KONFIGURASI SERVICE IP

Setelah mempelajari berbagai jenis SERVICE IP pada MikroTik RouterOS dan cara mengonfigurasinya, pada bagian ini kita akan melakukan pengujian langsung terhadap perubahan konfigurasi yang telah dilakukan. Pengujian ini bertujuan untuk memverifikasi bahwa perubahan konfigurasi SERVICE berhasil diterapkan dan layanan masih dapat diakses dengan benar.

7.1 Pengujian Perubahan Port Winbox

Mengubah port default Winbox dari 8291 menjadi 2025

Pada pengujian ini, kita akan mengubah port default Winbox dari 8291 menjadi 2025, kemudian menguji akses ke router menggunakan port baru tersebut.

Langkah 1: Mengubah port Winbox menggunakan CLI dengan perintah berikut:

/ip service print
# Melihat daftar SERVICE saat ini dan port yang digunakan
/ip service set winbox port=2025
# Mengubah port Winbox dari 8291 menjadi 2025
/ip service print
# Memverifikasi perubahan port Winbox
Mengakses MikroTik dengan Winbox menggunakan port 2025

Pada gambar di atas, kita dapat melihat bahwa untuk terhubung ke router menggunakan Winbox dengan port yang telah diubah, kita perlu menambahkan :2025 setelah alamat IP router (contoh: 192.168.88.1:2025).

Sangat penting untuk mencatat port baru yang digunakan setelah melakukan perubahan. Jika Anda lupa port yang telah diubah, Anda mungkin tidak dapat mengakses router hingga Anda melakukan reset atau mengakses melalui metode lain seperti koneksi serial.

7.2 Pengujian Pembatasan Akses SERVICE SSH

Membatasi akses SSH hanya untuk alamat IP tertentu

Pada pengujian ini, kita akan membatasi akses SSH hanya untuk jaringan 192.168.88.0/24, yang berarti hanya perangkat dalam jaringan lokal yang dapat mengakses router melalui SSH.

Langkah 1: Mengonfigurasi pembatasan akses SSH menggunakan CLI:

/ip service set ssh address=192.168.88.0/24
# Membatasi akses SSH hanya dari jaringan 192.168.88.0/24
/ip service print where name="ssh"
# Memverifikasi perubahan pada SERVICE SSH

Langkah 2: Melakukan pengujian akses SSH dari alamat IP yang diizinkan dan yang tidak diizinkan.

Akses SSH berhasil dari alamat IP yang diizinkan (192.168.88.10)

Dari hasil pengujian di atas, dapat dilihat bahwa pembatasan akses SSH berhasil diterapkan. Perangkat dengan alamat IP dalam jaringan 192.168.88.0/24 dapat terhubung melalui SSH, sementara perangkat dengan alamat IP di luar jaringan tersebut tidak dapat terhubung.

Pembatasan akses berdasarkan alamat IP adalah lapisan keamanan penting, tetapi tidak boleh menjadi satu-satunya pertahanan. Kombinasikan dengan kata sandi yang kuat, otentikasi kunci publik, dan pertimbangkan untuk mengubah port default SSH untuk meningkatkan keamanan.

7.3 Pengujian Konfigurasi SERVICE TELNET

Mengonfigurasi SERVICE telnet dengan pembatasan akses dan port alternatif

Pada pengujian ini, kita akan mengonfigurasi SERVICE telnet dengan mengubah port default dan membatasi akses, kemudian menguji koneksi ke router menggunakan konfigurasi baru tersebut.

Langkah 1: Mengonfigurasi SERVICE telnet dengan CLI:

/ip service print where name="telnet"
# Melihat konfigurasi telnet saat ini
/ip service set telnet port=8023 address=192.168.88.0/24
# Mengubah port telnet dari 23 menjadi 8023 dan membatasi akses ke jaringan lokal
/ip service print where name="telnet"
# Memverifikasi perubahan pada SERVICE telnet

Langkah 2: Menguji koneksi telnet menggunakan port dan alamat yang telah dikonfigurasi.

Mengakses MikroTik dengan telnet menggunakan port 8023

Pada gambar di atas, kita dapat melihat contoh mengakses router melalui telnet menggunakan port alternatif 8023. Perintah yang digunakan adalah telnet 192.168.88.1 8023 dari komputer dengan alamat IP dalam jaringan 192.168.88.0/24.

Meskipun kita telah mengonfigurasi telnet dengan port alternatif dan pembatasan alamat IP, perlu diingat bahwa telnet tetap mengirimkan kredensial dan data dalam bentuk teks biasa (tidak terenkripsi). Untuk keamanan yang lebih baik, sebaiknya gunakan SSH sebagai alternatif yang lebih aman untuk akses command line jarak jauh. Contoh ini ditunjukkan hanya untuk tujuan pembelajaran.

Mengubah port default dan membatasi akses berdasarkan alamat IP menambahkan lapisan keamanan dasar untuk layanan telnet. Namun, ini tidak mengatasi kelemahan fundamental telnet yaitu transmisi data yang tidak terenkripsi. Dalam lingkungan produksi, lebih baik menonaktifkan telnet sepenuhnya dan hanya menggunakan SSH.
