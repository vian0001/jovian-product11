1. MENGELOLA LAYANAN PADA MIKROTIK ROUTEROS

Dari sumber NetworkTik (2024), MikroTik RouterOS memiliki layanan yang secara default selalu diaktifkan. Layanan-layanan tersebut memiliki port yang ditugaskan untuk mereka dan dapat menjadi jalur bagi para peretas untuk menyerang router melalui port tersebut.

Oleh karena itu, selalu direkomendasikan untuk menonaktifkan layanan yang tidak penting, dan mengubah nomor port dari layanan yang ingin kita gunakan. Apa saja layanan tersebut, di mana kita dapat menemukannya, dan bagaimana cara memanipulasinya?

Untuk melihat layanan yang tersedia pada RouterOS, gunakan perintah berikut:

/ip service print
# Menampilkan daftar semua layanan yang tersedia

Dari sumber System Zone (2021), secara default MikroTik RouterOS mengaktifkan beberapa layanan login untuk mengakses RouterOS untuk tujuan administrasi. Layanan-layanan ini meliputi api, ftp, ssh, telnet, winbox, dan www.

Untuk menonaktifkan layanan yang tidak digunakan, gunakan perintah berikut:

/ip service disable telnet,ftp
# Menonaktifkan layanan telnet dan ftp

Atau dengan menentukan nomor layanan:

/ip service disable numbers=1,2
# Menonaktifkan layanan dengan nomor 1 dan 2

Dari sumber MikroTik Documentation (2024), RouterOS memiliki opsi bawaan untuk akses manajemen yang mudah ke perangkat jaringan. Layanan tertentu harus dimatikan pada jaringan produksi: MAC-Telnet, MAC-WinBox, dan MAC-Ping:

/tool mac-server set allowed-interface-list=none
/tool mac-server mac-winbox set allowed-interface-list=none
/tool mac-server ping set enabled=no

Dari sumber MikroTik Documentation (2023), meskipun firewall melindungi router dari antarmuka publik, Anda mungkin masih ingin menonaktifkan layanan RouterOS. Sebagian besar alat administratif RouterOS dikonfigurasi di menu /ip service.

Untuk mengubah port default layanan dan meningkatkan keamanan:

/ip service set ssh port=2222
# Mengubah port SSH dari 22 menjadi 2222
/ip service set www port=8081
# Mengubah port HTTP dari 80 menjadi 8081
/ip service set winbox port=9090
# Mengubah port Winbox dari 8291 menjadi 9090

Dari sumber MikroTik Documentation (2024), RouterOS mungkin memiliki layanan lain yang diaktifkan (mereka dinonaktifkan dalam konfigurasi RouterOS default). Proxy caching MikroTik, socks, UPnP, dan layanan cloud:

/ip proxy set enabled=no
/ip socks set enabled=no
/ip upnp set enabled=no
/ip cloud set ddns-enabled=no update-time=no

Dari sumber System Zone (2021), beberapa administrator MikroTik RouterOS memodifikasi port layanan login dengan port yang tidak digunakan. Namun ini memiliki kerugian karena Anda mungkin lupa nomor port yang dimodifikasi dan tidak dapat masuk ke Router MikroTik sampai mengingat nomor port yang ditetapkan. Lagi pula, sangat mudah bagi peretas untuk menemukan port dan layanan yang dimodifikasi menggunakan beberapa alat penemuan jaringan dan layanan.

Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa mengelola layanan pada MikroTik RouterOS adalah langkah penting dalam mengamankan perangkat. Praktik terbaik meliputi menonaktifkan layanan yang tidak digunakan, mengubah port default untuk layanan yang digunakan, dan membatasi akses ke layanan tersebut. Meskipun mengubah port default dapat memberikan lapisan keamanan tambahan, ini bukanlah solusi lengkap karena port yang dimodifikasi masih dapat ditemukan oleh penyerang menggunakan alat pemindaian.

2. MEMBATASI AKSES KE LAYANAN MIKROTIK

Dari sumber System Zone (2021), selalu merupakan praktik yang baik untuk mengaktifkan akses layanan login dari alamat IP tertentu. Jika kita menerapkan akses layanan login berdasarkan alamat IP, peretas dari WAN akan selalu ditolak untuk mengakses layanan login dan karenanya, RouterOS akan lebih aman.

Untuk membatasi akses ke layanan berdasarkan alamat IP, gunakan perintah berikut:

/ip service set ssh address=192.168.88.0/24
# Membatasi akses SSH hanya dari jaringan 192.168.88.0/24
/ip service set winbox address=192.168.88.0/24,10.0.0.5
# Membatasi akses Winbox dari jaringan 192.168.88.0/24 dan alamat 10.0.0.5

Dari sumber LinkedIn (2023), cara lain untuk menambahkan lapisan keamanan adalah dengan membuat pengguna baru untuk penggunaan sehari-hari dan menonaktifkan akun pengguna admin, atau alternatifnya Anda dapat mengubah nama pengguna dari pengguna default dari admin ke nama pengguna kustom. Batasi permukaan serangan dengan mematikan layanan dan port yang tidak digunakan. Layanan seperti ssh, ftp, telnet, dan web dapat digunakan oleh penyerang untuk membrute force Mikrotik Anda.

Untuk membuat pengguna baru dengan hak admin dan menonaktifkan pengguna admin default:

/user add name=superadmin group=full password=StrongP@ssw0rd
# Membuat pengguna baru dengan hak penuh
/user disable admin
# Menonaktifkan pengguna admin default

Dari sumber MikroTik Documentation (2024), mengubah nama pengguna default admin ke nama yang berbeda. Nama kustom membantu melindungi akses ke router Anda jika seseorang memiliki akses langsung ke router Anda. Router MikroTik memerlukan konfigurasi kata sandi, kami sarankan menggunakan alat pembuat kata sandi untuk membuat kata sandi yang aman dan tidak berulang.

Dari sumber SIM-Cloud (2023), untuk keamanan yang lebih tinggi, kami merekomendasikan agar Anda masuk ke Mikrotik sebagai pengguna unik. Untuk melakukan ini, buat pengguna baru, tentukan kata sandi untuknya dan tambahkan ke grup dengan hak administrator.

Jangan gunakan nama yang umum digunakan seperti 'user', 'guest', 'admin', dll. Jangan gunakan kata sandi seperti '12345', 'qwerty' atau sejenisnya dan jangan gunakan nama pengguna Anda lagi sebagai kata sandi. Kata sandi harus tidak kurang dari 8 karakter dan harus menyertakan huruf kecil dan huruf besar, angka, dan idealnya karakter khusus.

Contoh membuat pengguna baru dengan nama dan kata sandi yang kuat:

/user add name=johndoe@admin password=C0mpl3x#P4ss group=full
# Membuat pengguna dengan nama unik dan kata sandi kompleks

Kesimpulan: Dari 3 pendapat tersebut dapat disimpulkan bahwa membatasi akses ke layanan MikroTik adalah strategi penting dalam keamanan jaringan. Ini mencakup membatasi akses layanan berdasarkan alamat IP, membuat pengguna dengan nama unik dan kata sandi yang kuat, serta menonaktifkan pengguna admin default. Dengan membatasi siapa yang dapat mengakses layanan dan dari mana, risiko akses tidak sah ke router dapat dikurangi secara signifikan.

3. MENGAMANKAN ROUTER DENGAN ATURAN FIREWALL

Dari sumber Tech@Layer-x (2024), Mengkonfigurasi Firewall MikroTik sangat penting untuk menjaga keamanan dan kinerja jaringan. Aturan firewall yang diatur dengan benar dapat melindungi jaringan Anda dari akses tidak sah dan berbagai ancaman cyber. Panduan ini menguraikan praktik terbaik untuk mengkonfigurasi firewall MikroTik untuk mengoptimalkan keamanan dan efisiensi.

Urutan aturan firewall secara signifikan memengaruhi efektivitasnya, karena MikroTik memproses aturan secara berurutan dari atas ke bawah.

Berikut adalah contoh aturan firewall dasar untuk mengamankan router MikroTik:

# Membuat daftar antarmuka
/interface list add name=WAN
/interface list add name=LAN
/interface list member add interface=ether1 list=WAN
/interface list member add interface=ether2 list=LAN
# Aturan firewall input - melindungi router
/ip firewall filter add chain=input connection-state=established,related action=accept comment="Izinkan koneksi established/related"
/ip firewall filter add chain=input in-interface-list=LAN action=accept comment="Izinkan akses dari LAN"
/ip firewall filter add chain=input protocol=icmp action=accept comment="Izinkan ping"
/ip firewall filter add chain=input action=drop comment="Drop semua koneksi masuk lainnya"
# Aturan firewall forward - melindungi jaringan
/ip firewall filter add chain=forward connection-state=established,related action=accept comment="Izinkan koneksi established/related"
/ip firewall filter add chain=forward in-interface-list=LAN action=accept comment="Izinkan akses keluar dari LAN"
/ip firewall filter add chain=forward action=drop comment="Drop semua koneksi forward lainnya"

Dari sumber MikroTik Documentation (2024), dari semua yang telah kita pelajari sejauh ini, mari kita coba membangun firewall yang canggih. Dalam contoh pembangunan firewall ini, kita akan mencoba menggunakan sebanyak mungkin fitur firewall untuk mengilustrasikan bagaimana cara kerjanya dan kapan mereka harus digunakan dengan cara yang benar. Sebagian besar penyaringan akan dilakukan dalam firewall RAW, firewall reguler akan berisi hanya satu set aturan dasar untuk menerima koneksi established, related, dan untracked serta menjatuhkan semua yang lain yang tidak berasal dari LAN untuk sepenuhnya melindungi router.

Berikut adalah contoh konfigurasi firewall tingkat lanjut menggunakan fitur RAW:

# Aturan firewall RAW untuk pemfilteran awal
/ip firewall raw add chain=prerouting in-interface-list=WAN protocol=tcp tcp-flags=syn connection-state=new action=drop comment="Anti-Spoofing: Drop Invalid SYN"
/ip firewall raw add chain=prerouting in-interface-list=WAN src-address=192.168.0.0/16 action=drop comment="Anti-Spoofing: Drop private addresses from WAN"
/ip firewall raw add chain=prerouting in-interface-list=WAN src-address=10.0.0.0/8 action=drop comment="Anti-Spoofing: Drop private addresses from WAN"
/ip firewall raw add chain=prerouting in-interface-list=WAN src-address=172.16.0.0/12 action=drop comment="Anti-Spoofing: Drop private addresses from WAN"

Dari sumber Manito Networks (2018), koneksi manajemen ke perangkat jaringan harus dibuat menggunakan protokol aman dengan modul kriptografi yang divalidasi FIPS 140-2. Meskipun konfigurasi telah diubah, penyerang masih dapat mencoba masuk dengan kredensial admin default Mikrotik melalui Internet menggunakan SSH atau Winbox dan akan berhasil. Kita akan menonaktifkan akses (input) dari sisi WAN dengan menambahkan beberapa aturan firewall dasar, lalu membuat daftar alamat firewall untuk memblokir Bogons.

Kita akan menonaktifkan akses (input) dari sisi WAN dengan menambahkan beberapa aturan firewall dasar, lalu membuat daftar alamat firewall untuk memblokir Bogons. Bogons adalah alamat jaringan yang tidak memiliki urusan untuk diarahkan melalui Internet, dan biasanya tidak diarahkan kecuali jika seseorang mencoba memalsukan lalu lintas dan membuatnya seolah-olah berasal dari jaringan Anda sendiri.

Contoh konfigurasi untuk memblokir Bogons:

# Membuat daftar alamat untuk Bogons
/ip firewall address-list add list=bogons address=0.0.0.0/8 comment="RFC 1700 - IANA reserved"
/ip firewall address-list add list=bogons address=127.0.0.0/8 comment="RFC 1700 - Loopback"
/ip firewall address-list add list=bogons address=224.0.0.0/4 comment="RFC 1700 - Multicast"
/ip firewall address-list add list=bogons address=240.0.0.0/4 comment="RFC 1700 - Reserved"
# Menerapkan filter Bogons
/ip firewall filter add chain=input in-interface-list=WAN src-address-list=bogons action=drop comment="Drop Bogons from WAN"
/ip firewall filter add chain=forward in-interface-list=WAN src-address-list=bogons action=drop comment="Drop Bogons from WAN (Forward)"

Kesimpulan: Dari 3 pendapat tersebut dapat disimpulkan bahwa mengamankan router MikroTik dengan aturan firewall adalah langkah kritis dalam melindungi infrastruktur jaringan. Pendekatan terbaik adalah menerapkan prinsip "izinkan yang diizinkan" daripada "blokir yang dilarang", dengan fokus pada penyaringan di tingkat RAW untuk efisiensi, memblokir alamat Bogons, dan menerapkan aturan berdasarkan status koneksi. Aturan firewall harus diatur dengan benar sesuai urutan prioritas untuk memastikan keamanan maksimal tanpa mengganggu lalu lintas yang sah.

4. PRAKTIK TERBAIK KEAMANAN TAMBAHAN

Dari sumber MikroTik Documentation (2024), langkah-langkah berikut adalah rekomendasi tentang cara melindungi perangkat Anda dengan aturan firewall yang kuat yang sudah dikonfigurasi. Mulailah dengan meningkatkan versi RouterOS Anda. Beberapa rilis lama memiliki kelemahan atau kerentanan tertentu, yang telah diperbaiki. Jaga perangkat Anda tetap up to date, untuk memastikan aman.

Perintah untuk memeriksa dan mengupgrade RouterOS:

/system package update check-for-updates
# Memeriksa pembaruan
/system package update download
# Mengunduh pembaruan
/system package update install
# Menginstal pembaruan

Dari sumber MikroTik Documentation (2024), untuk mencegah akses jarak jauh ke perangkat Anda, ada firewall yang telah dikonfigurasi sebelumnya yang memblokir koneksi WAN (sisi internet). Ini disengaja, jangan hapus aturan ini kecuali Anda benar-benar yakin bahwa koneksi aman. Jika Anda berniat membuka akses jarak jauh ke perangkat Anda, kami merekomendasikan mengamankan koneksi menggunakan Virtual Private Network (VPN) seperti WireGuard.

Contoh konfigurasi dasar WireGuard VPN:

# Membuat interface WireGuard
/interface wireguard add name=wireguard1 listen-port=13231
/interface wireguard print
# Catat private-key dan public-key yang dihasilkan
# Menambahkan peer WireGuard
/interface wireguard peers add interface=wireguard1 public-key="CLIENT_PUBLIC_KEY" allowed-address=10.10.10.2/32
# Menambahkan alamat IP ke interface WireGuard
/ip address add address=10.10.10.1/24 interface=wireguard1
# Mengizinkan koneksi WireGuard melalui firewall
/ip firewall filter add chain=input protocol=udp dst-port=13231 action=accept comment="Allow WireGuard"

Dari sumber LinkedIn (2023), penggunaan aturan firewall di Mikrotik menyediakan banyak kustomisasi dan fitur keamanan termasuk: Pemfilteran paket untuk menentukan bagaimana paket harus diproses saat mereka melewati router dan NAT (Network Address Translation) yang memungkinkan alamat pribadi dipetakan ke satu alamat publik atau beberapa alamat.

Kegagalan untuk mengatur aturan firewall mengekspos jaringan Anda ke berbagai jenis serangan termasuk DDoS dan akses tidak sah. Mengizinkan semua lalu lintas masuk dan keluar tanpa penyaringan dapat menyebabkan potensi kerentanan keamanan.

Konfigurasi NAT untuk jaringan:

# Konfigurasi NAT untuk akses internet
/ip firewall nat add chain=srcnat out-interface-list=WAN action=masquerade comment="NAT untuk koneksi keluar"

Dari sumber NeuronVM (2025), dalam dunia digital saat ini, keamanan adalah topik penting, dan aturan Firewall adalah garis pertahanan pertama kita; aturan firewall seperti penjaga gerbang yang bekerja untuk menjaga data penting Anda tetap aman, ketika datang ke firewall, router MikroTik dikenal karena menyediakan sistem yang sangat kuat dan sangat dapat disesuaikan. Router MikroTik dikenal karena penuh dengan fitur, dan sebagian besar dari itu berasal dari firewall yang kuat, firewall MikroTik melacak status setiap koneksi, yang memungkinkan Anda membuat keputusan yang lebih cerdas tentang lalu lintas apa yang harus diizinkan dan apa yang harus diblokir.

Default Deny adalah prinsip keamanan utama, Default Deny memblokir semua kecuali jika Anda secara khusus mengatakan itu diizinkan; itu membuat jaringan Anda jauh lebih aman, firewall MikroTik sempurna untuk ini, memberi Anda alat untuk membangun fondasi yang aman untuk jaringan Anda.

Contoh konfigurasi backup otomatis untuk keamanan:

# Membuat skrip backup otomatis
/system script add name=auto-backup source={
:local backupfile ([/system identity get name] . "-" . \
[:pick [/system clock get date] 4 6] . \
[:pick [/system clock get date] 0 3] . \
[:pick [/system clock get date] 7 11] . ".backup")
/system backup save name=$backupfile
}
# Menjadwalkan backup mingguan
/system scheduler add name=weekly-backup interval=7d on-event=auto-backup start-time=03:00:00

Kesimpulan: Dari 3 pendapat tersebut dapat disimpulkan bahwa praktik terbaik keamanan tambahan untuk MikroTik meliputi memperbarui RouterOS secara teratur, menggunakan VPN untuk akses jarak jauh, menerapkan prinsip Default Deny pada firewall, dan melakukan backup konfigurasi secara berkala. Dengan menerapkan pendekatan keamanan berlapis, router MikroTik dapat dilindungi dari berbagai ancaman siber, memastikan integritas dan ketersediaan jaringan.
